INTRODUCTION


HTML is the standard markup page for the creation of Web pages. HTML stands for Hypertext Markup Language. HTML describes the structure of a Web page and consists of a series of elements. Overall, HTML tells the browser how to display the content on the web page.


ELEMENTS

HTML elements are the contents stored inside the start and end tags. Note: there are some elements that don't require end tags (like "br"), which are also known as empty elements. All HTML pages should be nested HTML elements, which is when elements contain other elements. For example, the "html" tag must contain "head" and "body" starting and ending tags inside the "html" tag.
Some HTML tags don't require an ending tag, but it is always recommended to include them to avoid errors and bugs. HTML is not case sensitive, but it is recommended to use lowercases, at least when it comes to using tags.

For the list of available HTML tags, visit HTML Tags .


ATTRIBUTES

HTML attributes provides additional information to HTML elements. This is available to all HTML elements and are specified in the starting tag. Examples of attributes are "href" (used in hyperlinks), src (used in images), "width" and "height" (used in images), "alt" (used in images"), "style" (used for CSS styling), "lang" (used to declare the default language of the Web page, which can be found in the source code for the "html" starting tag ), "title", etc.
HTML attributes aren't required to be lowercased and surrounded by double quotes, but it is recommended to do so. Single quotes are also allowed in some situations.

BLOCK AND INLINE

Every HTML element has a display value. However, in HTML, there are two types of display values. These two types are block and inline.
Block-level elements are elements that alawys start on new lines along with the browser adding a margin before and after the element. Block-level elements typically takes up the full width of the display. The most commonly used block elements are the "p" and "div" tags. Other block elements are the "address", "article", "aside", "blockquote", "canvas", "dd", "div", "dl", "dt", "fieldset", "figcaption", "figure", "footer", "form", "h1-h6", "header", "hr", "li", "main", "nav", "noscript", "ol", "p", "pre", "section", "table", "tfoot", "ul", and "video" tags.
Inline elements are elements that doesn't start on a new line. Unlike block elements, inline elements will only take as much width as it needs. Inline elements in HTML are the "a", "abbr", "acronym", "b", "bdo", "big", "br", "button", "cite", "code", "dfn", "em", "i", "img", "input", "kbd", "label", "map", "object", "output", "q", "samp", "script", "select", "small", "span", "strong", "sub", "sup", "textarea", "time", "tt", and "var" tags.
The "div" element is used in HTML as a container to divide different HTML elements. Attributes aren't required, although, its most common attributes are "id", "class", and "style". Below is an example of the "style" attribute used on a "div" element.

HEAD

The HTML "head" element contains the meta information of the HTML page. It is a container for the "title", "style", "meta", "link", "script", and the "base" element. The "head" element is located in the HTML code between the "html" tag and the "body" tag. The information inside the "head" tags will not be displayed in the HTML page. The "head" section is responsible for defining the document's title, character set, styles, scripts, and/or any other meta information that the user wishes to use in his/her page.
The "title" element allows the user to define the title of the HTML document. The title must be text only. It will be displayed in the browser's title bar or the page's tab. The title is a required element for HTML. The page title is important for Search Engine Optimization (SEOs) since the title is used to decide the order when listing pages in search results. Overall, the "title" element defines a title for the browser's toolbar, provides a title for the page (in case the user stores the HTML site in the Favorites folder), and it displays a title for the page in search engines.
The "style" element is used to define internal CSS styling information for the HTML page.
The "link" element links external files into the current HTML file through the "link" tag.
The HTML "meta" element is used to specify the character sets, page descriptions, keywords, author of the document, and viewport settings. It will not be displayed on the pages, but the "meta" will be used by browsers (to display and reload content), search engines (particularly keywords), and other web services if necessary.
The viewports is the region of area where the user can view on their device. Viewports can vary, based on the user's device whether it's a computer, laptop, tablet, smartphone, etc. The "viewports" attriubte allows the browser to receive instructions on how the web page should be scaled and lays out the dimensions. The "width=device-width" sets the width for the page in order to follow the built in screen width of the user's device. The "initial- scale=1.0" sets the zoom level when the page loads into the browser.
The HTML "script" element implements client-side Javascripts.

See Javascript section to learn more about Javascript in HTML.

STYLE GUIDE

It's essential to have clean and organized HTML code to make it easy to read and understand. This section goes over many different ways to keep HTML code neat. Firstly, use lowercase element names since mixing uppercase and lowercase is confusing, developers normally use lowercase, it looks cleaner, and it's easier to use. Always declare the document type in the first line of the document and its corresponding tags below. Make sure to close all HTML elements, even if it may work without ending tags. Also, use lowercase attribute names and quote attribute values since developers do it, it makes the value easier to read, and it won't take the value if there is a space in between. Always specify height, width, and alt for images to reduce flickering and creates alternate text if the image can't load into the page. HTML allows spaces around equal signs but no spaces is easier to read and groups entities better. Avoid using long code lines so the user doesn't have to scroll left and right to view the code. Don't add blank lines, spaces, or indentations for no reason. Adding blank lines is useful however for seperating large code blocks and two spaces of indentation is good enough for seperating entities. Never skip the title element because it is a requirement for HTML since SEOs use the page title using search engine algorithms to order pages in search results. HTML and body tags are strongly recommended to be added, even if the page will still work since it can cause errors for old browsers and can possibly crash the DOM and XML software. The same goes for the head tag, but there will be no consequences if the head tag isn't included since the browser will automatically create a head section for code before the body. It is optional to close empty elements, but it's still recommended to close them in the chance of using XML or XHTML. You should always include the lang attribute to declare the official language of the HTML page to assist search engines and browsers. For proper interpretation and search engine indexing, the charset should be defined as early as possible in the HTML code. The viewport should be included since it sets the user's visible area of the page whether it's a small screen like a phone or big as a monitor or TV using the meta tag. Comments should be included when necessary to explain certain parts of the code for viewers to understand and it's recommended to use two spaces of indentation for comments. Use simple syntax when referencing external style sheets. For CSS styling in HTML pages, short CSS code can be compressed into one line but for long CSS code, write them using multiple lines with the styling code in between the brackets having two spaces of indentation. The same goes for Javascripts, with its own additional styling found in the Javascript portion of the HTMLEx program. Use lowercase file names since some web servers are case sensitive. HTML files should have an .html extension, CSS files should have a .css extension, and Javascript files should have a .js extension. Lastly, when a URL doesn't have a specified name, the server creates a default filename and it's important to keep note of the filename that is used for your HTML project.

ENTITIES

HTML reserved characters must be replaced with character entities. For example, the user may not be able to use the less than or greater than symbols since the browser may mistake them for the starting or ending tag. Therefore, character entities are used to display HTML reserved characters in HTML pages. To display a less than symbol, it will be displayed as < or < see source code . The advantage of using an entity name is that it's easy to remember while the disadvantage is that browsers may not support all entity names since they mostly support entity numbers. A common entity in HTML is the non-breaking space ( );, which is a space that doesn't break into a new line. This is useful when the user doesn't want to break any words as well as it prevents browsers from spacing in HTML pages. It's important to note that entity names are case sensitive.
Diacritical marks are "glyphs" added to a letter. They can appear ablove or below a letter, inside, and/or in between two letters. Diacritical marks can be used with alphanumeric characters to produce characters that aren't typically in the character set. An example of this is à.

URL ENCODE

Uniform Resource Locators (URLs) are web addresses composed of words or Internet Protocol (IP) addresses. Web browsers request pages from web servers using a URL. URLs addresses documents on the web. The format of a URL is scheme://prefix.domain:port/path/filename . Scheme defines the type of Internet service used (most commmonly http or https). Prefix defines the domain prefix (most commonly www.). Domain defines the Internet domain name. Port defines the port number at the host (default for http is 80). Path defines the path or root directory for the server. Filename defines the name of a document/resource. Common URL schemes include http (used for common web pages unencrypted), https (used for secure web pages encrypted), ftp (downloading/ uploading files), and file (file on your computer). URLs are sent over the Internet only using the ASCII characters. Otherwise, the URL will have to be converted. URL encoding converts non ASCII characters into a format that can be transmitted over the Internet by replacing non ASCII characters with a "%" followed with hexadecimal digits. URLs also can't contain spaces and encoding would replace a space character with a "+" or "%20". For HTML5, the default charset is UTF-8. For the complete reference of URL encoding, check URL Encoding

XHTML

XHTML is the stricter XML based version of HTML. XTHML stands for Extensible Hypertext Markup Language and it defines HTML as an XML application. XHTML is available for all major browsers. In XHTML, all documents must be marked up correctly or be "well-formed". XHTML was designed to make HTML more extensible and flexible to work with other data formats like XML. Browsers tend to ignore errors in HMTL pages and therefore displays the website despite errors, but in XHTML, it is much more stricter with error handling. The most important differences between XHTML and HTML when it comes to technical coding is that in XHTML, the DOCTYPE tag is mandatory, the "xmlns" attribute in the HTML tag is mandatory, the html, head, title, and body tags are mandatory, elements must be properly nested, elements must always be closed, elements and attributes must always be lowercased, attribute values must always be quoted, and attribute minimization is forbidden.
